# Smart Content Hub | منصة إدارة المحتوى الذكية

![Smart Content Hub Banner](https://via.placeholder.com/1200x400/007bff/ffffff?text=Smart+Content+Hub)

## 🚀 مقدمة

Smart Content Hub هي منصة SaaS متكاملة ومبتكرة مصممة لإحداث ثورة في طريقة إدارة المحتوى الرقمي. توفر هذه الأداة القوية للمستخدمين بيئة منظمة لتنظيم، تحليل، ونشر محتواهم بكفاءة عالية، مع الاستفادة من أحدث تقنيات الذكاء الاصطناعي لتقديم ميزات فريدة ومتقدمة. سواء كنت مدونًا، مسوقًا، أو فريق محتوى، فإن Smart Content Hub هي شريكك الأمثل لتبسيط سير عمل المحتوى وتحقيق أقصى تأثير.

## ✨ الميزات الرئيسية

### الميزات المجانية

*   **تنظيم المحتوى**: مساحات عمل مخصصة لإدارة المقالات، المنشورات، والوسائط المتعددة.
*   **تتبع الحالة**: تصنيف المحتوى حسب حالته (مسودة، منشور، مؤرشف) لتسهيل التتبع.
*   **واجهة مستخدم بديهية**: تصميم نظيف وسهل الاستخدام يضمن تجربة سلسة.

### الميزات المميزة (Premium) - مدعومة بالذكاء الاصطناعي

*   **أتمتة النشر الذكي**: جدولة ونشر المحتوى تلقائيًا عبر منصات متعددة بناءً على تحليلات الأداء.
*   **تحسين المحتوى بالذكاء الاصطناعي**: اقتراحات لتحسين صياغة المحتوى، الكلمات المفتاحية، وعناوين SEO لزيادة الوصول.
*   **تحليل الأداء المتقدم**: لوحات تحكم تفاعلية توفر رؤى عميقة حول أداء المحتوى وتفاعل الجمهور.
*   **إنشاء محتوى مساعد**: أدوات مدعومة بالذكاء الاصطناعي لمساعدتك في توليد أفكار المحتوى أو صياغة مسودات أولية.

## 💰 نموذج الربحية (Monetization)

تعتمد Smart Content Hub على نموذج اشتراك مرن (Subscription-as-a-Service) يوفر طبقات خدمة مختلفة لتلبية احتياجات متنوعة:

*   **الخطة المجانية**: وصول للميزات الأساسية لإدارة المحتوى.
*   **الخطة الأساسية (Basic)**: ميزات إضافية مثل أتمتة الجدولة وتحليلات بسيطة.
*   **الخطة المميزة (Premium)**: وصول كامل لجميع الميزات المدعومة بالذكاء الاصطناعي، التحليلات المتقدمة، ودعم الأولوية.

يتم التعامل مع جميع عمليات الدفع بأمان عبر **Stripe API**، مما يضمن تجربة دفع سلسة وموثوقة.

## 🛠️ المكدس التقني (Technical Stack)

تم بناء Smart Content Hub باستخدام أحدث التقنيات لضمان الأداء العالي، قابلية التوسع، وتكلفة استضافة صفرية:

*   **الواجهة الأمامية (Frontend)**:
    *   **React.js**: مكتبة JavaScript لبناء واجهات المستخدم التفاعلية.
    *   **Vite**: أداة بناء سريعة لتجربة تطوير محسّنة.
    *   **TypeScript**: لغة برمجة توفر كتابة قوية لزيادة موثوقية الكود.
    *   **Tailwind CSS**: إطار عمل CSS لإنشاء تصميمات سريعة ومتجاوبة.
*   **الواجهة الخلفية (Backend)**:
    *   **Supabase Edge Functions**: وظائف لا خادمية (Serverless Functions) تعتمد على Deno لمعالجة منطق الأعمال الحساس (مثل معالجة الدفع).
*   **قاعدة البيانات (Database)**:
    *   **Supabase (PostgreSQL)**: قاعدة بيانات علائقية سحابية توفر أيضًا مصادقة المستخدمين.
*   **الدفع (Payment)**:
    *   **Stripe API**: بوابة دفع آمنة ومرنة لإدارة الاشتراكات.
*   **الأتمتة (CI/CD)**:
    *   **GitHub Actions**: لأتمتة عمليات البناء، الاختبار، والنشر.
*   **الاستضافة**: 
    *   **GitHub Pages**: لاستضافة الواجهة الأمامية الثابتة مجانًا.

## 🚀 البدء السريع (Getting Started)

لإعداد المشروع وتشغيله محليًا، اتبع الخطوات التالية:

1.  **استنساخ المستودع (Clone the repository)**:
    ```bash
    git clone https://github.com/your-username/smart-content-hub.git
    cd smart-content-hub
    ```

2.  **تثبيت التبعيات (Install dependencies)**:
    ```bash
    pnpm install
    ```

3.  **إعداد متغيرات البيئة (Environment Variables)**:
    *   قم بإنشاء ملف `.env` في الجذر الرئيسي للمشروع.
    *   انسخ المحتوى من `.env.example` والصقه في ملف `.env`.
    *   احصل على مفاتيح Supabase و Stripe من لوحات التحكم الخاصة بهما وقم بتحديث القيم في ملف `.env`.
    ```env
    VITE_SUPABASE_URL=https://your-project-id.supabase.co
    VITE_SUPABASE_ANON_KEY=your-anon-key
    VITE_STRIPE_PUBLISHABLE_KEY=pk_test_your-publishable-key
    STRIPE_SECRET_KEY=sk_test_your-secret-key
    ```

4.  **تشغيل التطبيق محليًا (Run the application locally)**:
    ```bash
    pnpm run dev
    ```
    سيتم تشغيل التطبيق على `http://localhost:5173` (أو منفذ آخر متاح).

## ☁️ النشر (Deployment)

يتم النشر تلقائيًا باستخدام GitHub Actions عند كل `push` إلى الفرع الرئيسي (`main`).

### الواجهة الأمامية (Frontend)

*   يتم بناء الواجهة الأمامية ونشرها على **GitHub Pages**.
*   تأكد من تفعيل GitHub Pages في إعدادات المستودع الخاص بك واختيار فرع `gh-pages` كمصدر.

### الواجهة الخلفية (Supabase Edge Functions)

*   يتم نشر وظائف Supabase Edge Functions تلقائيًا.
*   **إعداد GitHub Secrets**: يجب عليك إضافة المتغيرات التالية كـ GitHub Secrets في مستودعك (Settings -> Secrets and variables -> Actions):
    *   `SUPABASE_ACCESS_TOKEN`: رمز الوصول الخاص بـ Supabase CLI.
    *   `SUPABASE_PROJECT_ID`: معرف مشروع Supabase الخاص بك.
    *   `STRIPE_SECRET_KEY`: مفتاح Stripe السري الخاص بك (يستخدم في Edge Functions).
    *   `VITE_SUPABASE_URL`: رابط مشروع Supabase الخاص بك.
    *   `VITE_SUPABASE_ANON_KEY`: مفتاح Supabase Anon Key الخاص بك.
    *   `VITE_STRIPE_PUBLISHABLE_KEY`: مفتاح Stripe القابل للنشر الخاص بك.

## 🔒 الأمن وحماية مفاتيح الـ API

نولي أهمية قصوى لأمن بياناتك ومفاتيح الـ API. يتم التعامل مع جميع المفاتيح الحساسة (مثل `STRIPE_SECRET_KEY` و `SUPABASE_ACCESS_TOKEN`) باستخدام **GitHub Secrets** و **متغيرات البيئة (Environment Variables)** في Supabase Edge Functions.

*   **GitHub Secrets**: تضمن عدم كشف المفاتيح في الكود المصدري أو سجلات GitHub Actions.
*   **متغيرات البيئة**: يتم تمرير المفاتيح بأمان إلى بيئة التشغيل للوظائف اللاخادمية، مما يمنع الوصول غير المصرح به.
*   **سياسات الأمان على مستوى الصف (Row Level Security - RLS)**: مطبقة في قاعدة بيانات Supabase لضمان أن المستخدمين يمكنهم الوصول إلى بياناتهم الخاصة فقط.

## 🔄 الأتمتة الذكية (CI/CD) باستخدام GitHub Actions

ملف `deploy.yml` في `.github/workflows/` يقوم بأتمتة سير عمل التطوير بالكامل:

*   **البناء (Build)**: بناء الواجهة الأمامية والوظائف اللاخادمية.
*   **الاختبار (Linting)**: فحص جودة الكود والالتزام بالمعايير.
*   **النشر (Deployment)**: نشر الواجهة الأمامية إلى GitHub Pages ونشر وظائف Supabase Edge Functions.

يتم تشغيل هذا السير العمل تلقائيًا عند كل `push` إلى الفرع الرئيسي أو عند فتح طلبات السحب، مما يضمن تحديثات سريعة وموثوقة.
